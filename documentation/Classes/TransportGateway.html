TransportGateway

public class TransportGateway: NSObject

A TransportGateway is responsible for creating, configuring and executing URLTransportable instances. A transport gateway represents a remote server.

  • defaultGateway

    The default TransportGateway instance. Most applications will only ever talk to one web server, if that is the case, you should configure and use this instance to create and execute URLTransportable instances.

    The baseURL of a transport gateway will be the baseURL of every URLTransportable created by this gateway’s makeRequest(identifiers:) method. This is typically the protocol and hostname of the server represented by this gateway, i.e., http://server.com.

  • token

    The gateway will use this token to validate whether or not request being queued by enqueueForAuthentication(_:) can be executed. This token must be non-nil and return true for isValid for the request to continue executing.

    If both the username and password are non-nil, the gateway will omit calling transportGateway:authenticateRequest:withToken: and fill out the Authorization header itself.

Methods
  • makeRequest(identifiers:)

    This method will create and return a new URLTransportable for configuration. The identifiers array can only be made up of URLValue instances. The request’s URL will be composed of this gateway’s baseURL followed by a path made up by joining these identifiers with the / character.

    For example, a gateway with the baseURL http://server.com that is sent makeRequest(identifiers:) with the argument ["users", 1] will produce an URLTransportable pointing at the URL http://server.com/users/1.

  • enqueueForAuthentication(_:)

    This method will take a configured URLTransportable previously returned by this gateway’s makeRequest(identifiers:) and, if the gateway believes it is authenticated, will immediately execute it. If the gateway doesn’t believe it is authenticated, it will queue this request until the gateway is given authentication details.
